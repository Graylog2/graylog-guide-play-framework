Graylog2 for Play

Bring Graylog2 to your Play 2 application by using the
access log and application log module and enjoy the power of centralized logging.

The Graylog2 module enables you to write both your Play access log as well as your application log to a Graylog2 server, in addition to the log files on your machines.

You can opt to use both features, or disable either one at will, if for example your are only interested in centralized access logs.

Access Log

The Play framework does not ship with an access log by default, making it difficult to analyze application usage in production.
Using this module you can easily get a centralized access log.

Application log

The Graylog2 Play module implements a standard logback appender, thereby making it easy to forward all of your application's logs to your Graylog2 system for easier analysis and debugging.

Add the dependency

In your appDependencies add the following new dependency:

"org.graylog2" % "play2-graylog2_2.10" % "1.1"

Add the resolver

In your build.sbt or Build.scala add the following new resolvers:

val main = play.Project(appName, appVersion, appDependencies).settings(
  resolvers += Resolver.url("Graylog2 Play Repository", url("http://http://graylog2.github.io/play2-graylog2/releases/"))(Resolver.ivyStylePatterns),
  resolvers += Resolver.url("Graylog2 Play Snapshot Repository", url("http://graylog2.github.io/play2-graylog2/snapshots/"))(Resolver.ivyStylePatterns)
)
